1) Overview

What it does

  • Upload a Job Description (paste text or file).
  • Upload multiple resumes (PDF / DOCX / TXT).
  • Scores combine 6 components: semantic similarity, keywords, must-haves, experience, education, bonus.
  • Shows matched keywords, missing must-haves, red flags, and strengths.
  • Makes final decisions: STRONG_HIRE, HIRE, MAYBE, NO_HIRE, REJECT.
  • Export ranked results to CSV.

Scoring Formula (v2.2.0)

35% semantic + 25% keywords + 20% must-haves + 10% experience + 5% education + 5% bonus

Must-haves are proportional (4/5 = 80%, not binary). Red flags apply -5% penalty each.

No training needed Fast & explainable C++/C#/.NET support

7) Using the App

Decision Types

  • STRONG_HIRE (85-100) ‚Äì Excellent fit, schedule interview immediately
  • HIRE (75-84) ‚Äì Good fit, move to next round
  • MAYBE (60-74) ‚Äì Borderline, needs review
  • NO_HIRE (45-59) ‚Äì Not a good fit
  • REJECT (0-44) ‚Äì Clear mismatch

Best Practices

  • Include clear "Requirements" section in JD
  • List 5-10 must-haves (not 30)
  • Use full names: "Python" not "Py"
  • Specify experience level if critical
  • Review "MAYBE" decisions manually

8) Troubleshooting

Common Issues

‚ùå "uvicorn not recognized"

Solution: Activate virtual environment or use python main.py

.venv\Scripts\activate
python main.py

‚ùå Nothing happens with python main.py

Solution: Ensure main.py has the startup block:

if __name__ == "__main__":
    import uvicorn
    uvicorn.run("main:app", host="127.0.0.1", port=8000, reload=True)

‚ùå Port 8000 already in use

Solution: Change port:

python -m uvicorn main:app --port 8001

‚ùå C++ or C# not matching

Solution: Ensure you're using v2.2.0 (check /api/health)

curl http://127.0.0.1:8000/api/health

Should return: {"version": "2.2.0"}

‚ùå Slow performance

Solution: Adjust config in main.py:

MAX_PARSE_WORKERS = 8  # Increase for more CPU cores
SEMANTIC_CHUNK_AGG = "max"  # Faster (slightly less accurate)

‚ùå Scanned PDFs not working

Solution: Add OCR support (optional):

pip install pytesseract
# Install Tesseract: https://github.com/UB-Mannheim/tesseract/wiki

9) Advanced Configuration

Tuning Performance

Edit these constants in main.py:

# Speed vs Accuracy
SEMANTIC_CHUNK_AGG = "weighted_top"  # or "max" (faster) or "avg"
SEMANTIC_TOP_K = 3  # Number of chunks to average

# File Limits
MAX_FILE_SIZE = 10 * 1024 * 1024  # 10MB
MAX_RESUMES = 50  # Max files per batch
MAX_PDF_PAGES = 100  # Pages to process per PDF

# Performance
MAX_PARSE_WORKERS = 4  # Parallel file parsing
FUZZY_PRE_FILTER = True  # Always keep enabled

Customizing Scoring Weights

Modify in main.py:

@dataclass
class ScoringWeights:
    semantic: float = 0.35    # Adjust these
    keywords: float = 0.25
    must_haves: float = 0.20
    experience: float = 0.10
    education: float = 0.05
    bonus: float = 0.05

Adding Custom Synonyms

Expand TECH_SYNONYMS in main.py:

TECH_SYNONYMS = {
    "your_term": ["synonym1", "synonym2"],
    # ... existing entries
}
